A) Folder Structure - top

I've separated hovers effects by type and you'll get 3 folders as following :

  • Borders
  • Overlays
  • Slides

The Borders folder contains the CSS markup with examples of effects on boders and using the CSS3 box shadow feature.

The Overlays folder is for effects displaying content above the images (texts, titles, other images or shadows).

The Slides folder contain all effects when the picture slides in any direction to reveal content (it can be another image or texts).

Some effects can be used at the same time and all those effects can be used simply by using the appropriate CSS class.

Those effects have been tested in all major browsers and due to the lack of support of CSS3 properties, some effects such as slidings or opacity won't work the same way in older browsers but the hidden content (for example in the Slides folder) will appear but there won't be any sliding transition. So the end user will still see all the content.

Here is the list of browsers supporting (almost) fully all effects :

  • Firefox 4 (Beta)
  • Chrome 4, 5, 6 and 7
  • Safari 4 and 5
  • Opera (latest version)

And here is the list of browsers which will display the hidden content on hover but without effects :

  • Internet Explorer 6, 7, 8 and 9
  • Firefox 2, 3, 3.5, 3.6

However, Firefox supports some CSS3 features but no transitions until the version 3.6

This list contains all browsers I've tested manually with native installations.

B) HTML Markup - top

In each HTML file, you'll find that all the content is wrapped in a "container", this is just for the demo and you don't need to use it. The same applies to the "clear" DIV after each list, it's just here to space elements.

I've used for all effects lists containing each time 3 items, sometimes the class containing the effect is applied to the UL element, sometimes to the LI element (or a nested DIV) but it would be the same using only DIVs.

BORDERS

The effects are simply applied to the image container, in all cases it's the LI element and the structure is always the same :

<ul>
    <li class="effect name"><img src="yourimage" alt="" />/li>
    <li class="effect name"><img src="yourimage" alt="" />/li>
    <li> ... </li>
</ul>

OVERLAYS

These effects are more complex and some content is revealed on mouse hover so here we have to separate the "default" and the "hidden" content.

Information Boxes

For this effect, there are 2 variants, the black and the white one. A surrounding background is appearing on mouse hover and shows some text under the image. In the following example, we see an image and a "details" DIV showing on hover, this DIV can contain text, a H2 heading title and a small paragraph in this example (you can change the "black" class by the "white" class) :

<ul>

    <li class="info">
    
        <a href="#"><img src="your image" alt="" /></a>
        
        <div class="details black">
            <h2>This is a title</h2>
            <p>Here comes a short description</p>
        </div>
        
    </li>

</ul>

The principle is the same for the "Bottom Overlay", "Top Overlay" and "Full Overlay" effects.

In this folder, there are 2 additional effects : the "Lomo" and "Image Stack". The first one works like the borders effects we have seen before, simply apply the "lomo" class to the parent container of the image. The "Image Stack" contains 2 images, the second one is showing above the first one on mouse hover. In this case, the "stack" effect is applied the the UL element containing the 2 LI, each one containing 1 image.

SLIDES

These effects work in the same way, the image is sliding in a specific direction to reveal some content is revealed on mouse hover so here we have to separate the "image" and the "hidden" content.

Again, we'll use here a UL element and we apply the "slide" class combined with another class which will specify in which direction the image will slide (moveup, moveleft, movedown, etc.)

Then this UL element will contain 2 LI elements which can contain an image or some text. Here is an example :

<ul class="slide moveright">

      <li><p>This text is revealed on mouse hover.</p></li>
      
      <li><img class="top" src="your image" alt="" /></li>

</ul>

In the above example, the image will slide to the right and reveal the paragraph text. Instead of a text, we can put a second image which will appear on mouse hover :

<ul class="slide moveup">

      <li><img class="bottom" src="your 2nd image" alt="" /></li>
      
      <li><img class="top" src="your 1st image" alt="" /></li>

</ul>

C) CSS Files and Structure - top

You'll find in each folder a "reset" CSS file, it is not necessary to use it but I would recommend you to do so, in that case it must be the very first CSS file linked from your HTML page.

Then you'll find all effects in "border.css", "slides.css" and "overlays.css" files. Each effect has a number in the HTML file and in the CSS so it's much easier for you to copy and paste the code you need.

All images used here have a 300 x 200px size, that means that when you find a specific width or height for an effect (all slides effects, the first border effect and most of the "overlays" effects), you have to change it according to the size of your image.

Otherwise, you may need to adjust colors according to your needs, keep in mind that in each CSS, the first part contain default styling used for this demo only (font, font size, typography), I've tried to keep it as simple as possible, now you'll find comments within the CSS to help you modify all effects according to your needs.
